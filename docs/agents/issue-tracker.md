# Issue tracker: GitHub

Issues and specifications live in GitHub Issues for this repository. Use `gh`
from this checkout to create, read, comment on, label, and close issues.

Pull requests are not a triage request surface.

When a skill says "publish to the issue tracker", create a GitHub issue. When
a skill says "fetch a ticket", read its full issue body and comments.

Blocking edges use GitHub issue dependencies when available; otherwise tickets
carry an explicit `Blocked by: #…` line.
